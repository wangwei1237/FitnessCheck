# FitnessCheck 验收测试计划 (Acceptance Test Plan)

为了确保 **FitnessCheck** 应用的功能符合预期并达到发布标准，以下是基于现有功能模块制定的验收性测试计划。

---

### 1. 计划与日历模块 (Dashboard & Calendar)

| ID | 测试场景 | 测试步骤 | 预期结果 |
| :--- | :--- | :--- | :--- |
| **TC-01** | **日历日期切换** | 在 Dashboard 页面点击周历中的不同日期。 | 1. 选中日期高亮显示。<br>2. 下方训练列表实时更新为对应日期的数据。 |
| **TC-02** | **新建训练计划** | 点击“新增训练计划”FAB -> 选择“胸部”肌群 -> 保存。 | 1. 弹出层消失。<br>2. 列表出现状态为“未开始”的胸部训练卡片。<br>3. 周历对应日期出现橙色状态点。 |
| **TC-03** | **周历翻页功能** | 点击周历头部的“左/右”箭头。 | 周历按周进行切换，且显示的月份/年份文字同步更新。 |
| **TC-04** | **回到今天** | 切换到非本周日期后，点击“今天”按钮。 | 1. 日历自动跳回包含当前日期的那一周。<br>2. 今天的日期被选中并展示。 |

### 2. 动作库模块 (Exercise Library)

| ID | 测试场景 | 测试步骤 | 预期结果 |
| :--- | :--- | :--- | :--- |
| **TC-05** | **动作分类筛选** | 进入动作库 -> 点击“背部”FilterChip。 | 列表仅显示属于“背部”的运动（如：引体向上）。 |
| **TC-06** | **自定义动作添加** | 点击“+”图标 -> 输入动作名“测试动作” -> 选择肌群 -> 确定。 | 1. 列表中出现带有“自定义”标记的新动作。<br>2. 重新进入应用后该动作依然存在。 |
| **TC-07** | **动作库注入模式** | 从训练记录页点击“添加动作” -> 动作库选中“杠铃卧推”。 | 1. 自动返回记录页。<br>2. 记录页成功增加“杠铃卧推”并默认开启第一组。 |

### 3. 训练记录核心模块 (Workout Logger)

| ID | 测试场景 | 测试步骤 | 预期结果 |
| :--- | :--- | :--- | :--- |
| **TC-08** | **开始/继续训练** | 点击首页“开始运动”或记录页“继续录入”。 | 1. 训练状态变更为 `IN_PROGRESS`。<br>2. 计时器开始运行。<br>3. 日历状态点变为蓝色。 |
| **TC-09** | **智能复刻 (Copy Set)** | 对某个动作点击“复制上一组”按钮。 | 1. 自动新增一组。<br>2. 新组的重量和次数与上一组完全一致。<br>3. 组序号正确递增。 |
| **TC-10** | **做组参数实时保存** | 修改某一组的重量为 80kg -> 退出页面 -> 重新进入。 | 1. 重量值保持为 80kg。<br>2. 修改过程无闪退或卡顿。 |
| **TC-11** | **完成训练流程** | 点击“结束训练” -> 输入心得备注 -> 保存。 | 1. 记录状态变为 `COMPLETED`。<br>2. 计时器停止并记录总时长。<br>3. 返回首页，该训练卡片显示“训练完成”且变灰。 |

### 4. 数据一致性与持久化 (Data & Persistence)

| ID | 测试场景 | 测试步骤 | 预期结果 |
| :--- | :--- | :--- | :--- |
| **TC-12** | **删除记录联动** | 删除首页的一个训练计划。 | 1. 首页列表移除卡片。<br>2. 日历上对应的状态指示点消失。<br>3. 数据库中该训练关联的所有 Set 也被级联删除。 |
| **TC-13** | **历史数据首组填充** | 昨天做过 100kg 卧推 -> 今天新建卧推记录并点击“添加”。 | 今天新添加的卧推第一组默认值自动填充为 100kg。 |
| **TC-14** | **冷启动数据保持** | 在后台杀掉 App 后重新打开。 | 所有已记录的训练、自定义动作、已完成状态均与杀进程前保持一致。 |

### 5. 用户体验与异常处理 (UX & Edge Cases)

| ID        | 测试场景         | 测试步骤                                     | 预期结果                                           |
|:----------|:-------------|:-----------------------------------------|:-----------------------------------------------|
| **TC-15** | **空状态展示**    | 切换到一个没有任何计划的日期。                          | 1. 页面显示“还没有训练计划”的空提示图。<br>2. 引导按钮“新增训练计划”功能正常。 |
| **TC-16** | **删除确认提醒**   | 在训练卡片上点击删除图标。                            | 弹出 AlertDialog 确认框，防止用户误触删除重要数据。               |
| **TC-17** | **多训练计划列表不遮挡** | 同一天创建至少 3 个训练计划，滚动到列表底部并点击最后一个计划的“开始运动”。 | 1. 最后一个计划可以滚动到悬浮“新增训练计划”按钮上方。<br>2. “开始运动”按钮完整可见且可点击。<br>3. 点击后进入对应训练记录页。 |

---

### 测试环境要求：
*   **设备**：Android 7.0 (API 24) 及以上版本的真机或模拟器。
*   **状态**：需包含预置的动作库数据（应用首次启动会自动植入）。
*   **工具**：可配合 `Logcat` 监控数据库操作是否报错。
